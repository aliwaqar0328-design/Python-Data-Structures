# Data Structures and Algorithms Mastery

This repository serves as a dedicated space for my journey through Data Structures and Algorithms (DSA), focusing on efficient problem-solving and clean code implementation in Python. As a student at COMSATS University, I have built this collection to bridge the gap between academic theory and practical interview preparation. The codebase is divided into specialized modules, currently focusing on various forms of Linked Lists, which are the building blocks for more complex systems.

## Singly Linked List Implementation
The implementation in `DSA(Linked List).py` covers the foundational mechanics of Singly Linked Lists, including standard operations like appending, prepending, and removing nodes. Beyond the basics, this module integrates several high-level algorithmic challenges frequently encountered in technical interviews. Notable features include a two-pointer approach to find the middle of a list, cycle detection logic, and a custom partition algorithm that reorganizes nodes around a specific value. I have also implemented a range-based reversal method, allowing for the inversion of specific segments within the list.

## Advanced Doubly Linked Lists
In the `DSA(DLL).py` file, the project expands into Doubly Linked Lists to leverage bidirectional traversal. This implementation is optimized for performance; for example, the `get` method is designed to determine whether to start searching from the head or the tail based on the target index, effectively halving the search time in many cases. This file also features solutions for complex structural problems, such as checking for palindromes, swapping the first and last elements without rebuilding the list, and a pair-swapping algorithm that rearranges adjacent nodes.

## How to Navigate this Repo
Each file is structured with a class-based implementation followed by a testing suite at the bottom. To see the algorithms in action, you can simply run the Python files and uncomment the method calls in the driver code. This repository is a living document, and I plan to continue adding more data structures like Stacks, Queues, and Trees as I progress through my curriculum and LeetCode practice.
